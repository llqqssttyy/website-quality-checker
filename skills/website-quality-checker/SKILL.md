# Website Quality Checker

Google Search Quality Rater Guidelines(QRG) 기반 웹페이지 품질 검사 스킬.

---

## 실행 흐름

### Phase 1: Discovery

1. 입력 수신 — URL이면 `web_fetch`로 가져오기. 실패 시 텍스트 붙여넣기 안내. 텍스트 직접 입력 시 평판(E)은 "확인 불가" 처리.
2. 페이지 유형 판별 — 정보성 | 상거래 | 블로그/개인 | 포럼/Q&A | 엔터테인먼트 | 정부/기관
3. YMYL 판별 — 명확한 YMYL(건강·금융·법률·안전·정부) / 중간(대학·부동산·육아) / 비YMYL(취미·유머·예술). 모호하면 `references/google-qrg-summary.md` 참조.
4. 가중치 확정 — YMYL 여부에 따라 조정. 상세: `references/scoring-weights.md`

### Phase 2: Analysis

6개 카테고리 순차 검사 (A → F), 각 0~100점 채점.

### Phase 3: Synthesis

5. `종합 점수 = Σ(카테고리 점수 × 가중치)` → 등급 부여 (S:90+ / A:75+ / B:55+ / C:35+ / D:0+)
6. 즉시 D등급 트리거 확인: A=0(유해) / C=0+YMYL(정보 전무) / E≤10(사기 평판) / F=0(스팸 확정)
7. `references/report-template.md` 참조하여 마크다운 보고서 생성

---

## 채점 원칙

모든 카테고리에 공통 적용하는 4가지 원칙:

**원칙 1: 증거 기반** — 점수는 콘텐츠에 명시적으로 존재하는 증거에 기반해서만 부여. 긍정 증거가 없는 항목은 60점 초과 불가. "~로 보인다", "~를 암시한다" 같은 추론은 증거가 아님. 판단 근거를 보고서에 반드시 명시.

**원칙 2: 검증 불가 = 보수적 점수** — 확인할 수 없는 항목은 중립(50)이 아닌 35~40점 부여. "확인할 수 없으니 신뢰할 수 없다"가 기본 입장.

**원칙 3: ✅/⚠️/❌ 환산** — ✅=만점, ⚠️=-10~15점, ❌=-20~30점. 100점에서 시작하여 감점. ❌ 1개 → 카테고리 상한 70점. ❌ 2개+ → 상한 50점.

**원칙 4: Devil's Advocate** — 6개 카테고리 채점 완료 후, 보고서 작성 직전에 자기 검증:

- 75점 이상 카테고리 → 증거 재확인. 부족하면 60점 이하로 하향.
- ✅ 판정 중 추론 기반이 있으면 ⚠️로 하향.
- "경쟁 사이트에서도 같은 점수를 줄 것인가?" / "QRG Rater가 납득할 등급인가?"

## 카테고리 A: 페이지 목적 & 유해성 (15%)

| #   | 항목             | 판단 기준                                             |
| --- | ---------------- | ----------------------------------------------------- |
| A1  | 목적 명확성      | "이 페이지가 왜 존재하는지" 바로 파악 가능한가        |
| A2  | 유익한 목적      | 정보 제공·엔터테인먼트·거래 등 사용자에게 도움 되는가 |
| A3  | 기만적 목적 아님 | 실제 목적을 숨기고 위장하지 않는가                    |
| A4  | 유해 콘텐츠 없음 | 자해/타해 조장, 혐오, 위험한 오해 유발이 없는가       |
| A5  | 제목-MC 일치     | 클릭베이트가 아닌가                                   |

**즉시 D등급**: A4 유해 확인 시, A3 명백한 기만 시. 세부: `references/lowest-quality-signals.md` 섹션 1~2.

## 카테고리 B: MC 품질 (25%)

| #   | 항목         | 판단 기준                                   |
| --- | ------------ | ------------------------------------------- |
| B1  | 노력         | 직접 제작 흔적 vs 자동 대량 생성            |
| B2  | 독창성       | 다른 곳에서 볼 수 없는 고유 정보·관점       |
| B3  | 재능/기술    | 페이지 목적에 맞는 수준으로 제작            |
| B4  | 정확성       | 사실적 오류 없음. YMYL이면 전문가 합의 일치 |
| B5  | MC 양 적절성 | 빈약하지도 필러로 채우지도 않음             |
| B6  | 필러 없음    | 의미 없는 반복·늘이기 없음                  |
| B7  | 제목-MC 일치 | 제목의 약속을 본문이 이행                   |

맥락별 가중: 정보성→B4, 블로그/리뷰→B2, 예술/창작→B3. 세부: `references/google-qrg-summary.md` 섹션 4.

## 카테고리 C: 사이트/작성자 정보 (10%, YMYL 시 13%)

| #   | 항목               | 판단 기준                                |
| --- | ------------------ | ---------------------------------------- |
| C1  | 운영 주체 명시     | About Us, 회사소개, 운영자 정보          |
| C2  | 작성자 정보        | 이름, 프로필, 약력                       |
| C3  | 연락처 정보        | 이메일, 전화, 문의 폼 (특히 상거래·YMYL) |
| C4  | 이해관계 충돌 없음 | 자사 제품 리뷰, 비공개 유료 협찬 아님    |

페이지 유형별 차등 감점 적용 (YMYL+상거래는 엄격, 취미 블로그는 관대). 상세 감점표: `references/scoring-weights.md` 섹션 3.

## 카테고리 D: E-E-A-T (25%, YMYL 시 28%)

| #   | 항목              | 판단 기준                                             |
| --- | ----------------- | ----------------------------------------------------- |
| D1  | Experience        | 1인칭 체험 서술, 직접 촬영 사진, 개인 에피소드        |
| D2  | Expertise         | 주제에 맞는 지식·자격. YMYL이면 전문 자격 필수        |
| D3  | Authoritativeness | 해당 분야에서 인정받는 사이트/작성자                  |
| D4  | Trust (핵심)      | 정확·정직·안전. 아무리 E-E가 높아도 Trust 없으면 낮음 |

YMYL 정보/조언 → D2 필수(자격 없으면 큰 감점). D2 상한표: `references/eeat-criteria.md` 섹션 5. 세부 판별 질문: `references/eeat-criteria.md`.

## 카테고리 E: 평판 (10%, YMYL 시 12%)

| #   | 항목                | 판단 기준                           |
| --- | ------------------- | ----------------------------------- |
| E1  | 외부 평판 확인 가능 | 뉴스 기사, 위키백과, 전문가 추천    |
| E2  | 긍정적 평판         | 수상, 전문가 인정, 상세한 긍정 리뷰 |
| E3  | 부정적 평판 없음    | 사기, 소비자 피해, 법적 제재 보도   |
| E4  | 작성자 평판         | 작성자 개인에 대한 외부 평가        |

조사: URL 있으면 `web_search`로 "[사이트명] review/scam/reputation" 검색. URL 없으면 35점(보수적). 검색 결과 정보 없으면 40점. **즉시 D등급**: E3에서 사기·범죄 평판 확인 시.

## 카테고리 F: 스팸/남용 탐지 (15%, YMYL 시 10%)

| #   | 항목                  | 판단 기준                                               |
| --- | --------------------- | ------------------------------------------------------- |
| F1  | 복사 콘텐츠 아님      | MC 전체가 다른 출처에서 복사되지 않음                   |
| F2  | AI 대량 생성 아님     | 노력·독창성 없이 AI로 찍어낸 콘텐츠 아님                |
| F3  | 사이트 평판 악용 없음 | 사이트 주제와 무관한 3자 스폰서 콘텐츠 아님             |
| F4  | 만료 도메인 악용 아님 | 도메인 이름과 콘텐츠 주제 불일치 아님                   |
| F5  | 해킹/변조/스팸 없음   | 해킹·스팸 댓글 도배 없음                                |
| F6  | 기만적 디자인 없음    | 광고 위장, 가짜 버튼, 가짜 검색 결과 없음               |
| F7  | 광고의 MC 방해 없음   | 광고가 MC를 가리거나 사용자 경험을 심각히 저해하지 않음 |

**즉시 D등급**: F1~F5 중 명확한 스팸/남용 확정 시. 세부: `references/lowest-quality-signals.md` 섹션 3.

## 개별 카테고리 검사

특정 카테고리만 요청 가능 (예: "E-E-A-T만 검사해줘" → D만 실행). 개별 검사 시 해당 카테고리만 보고서에 포함하고, 종합 점수는 "부분 검사"로 표기.
